# Backend guidance

- Read `context/current-state.md` before modifying the backend.
- Run Gradle commands from `backend/`.
- For ordinary backend changes, verify with `./gradlew test` on Unix-like systems or `gradlew.bat test` on Windows.
- Update `context/current-state.md` after meaningful backend changes.
- Add durable rules here only when they prevent a recurring mistake or capture repeated review feedback.
